# OSS 사이트 배포(인터넷에 올리기) 가이드 — Netlify 무료

> 작성: 2026-06-22 · 컴맹용 단계별. 사장님이 직접 하실 부분 + Claude가 도와줄 부분 표시.

## 0. 배포가 뭔가요?
지금 사이트는 **사장님 컴퓨터 + GitHub**에만 있어서 손님이 못 봐요.
"배포" = 인터넷에 올려서 **주소(예: `ossohayo.netlify.app`)를 만드는 것**.
- 무료예요.
- 한 번 연결해두면, 앞으로 Claude가 코드를 올릴 때마다 **자동으로 사이트가 갱신**돼요.

---

## ✅ 방법 A: GitHub 연결 (추천 — 한 번만 하면 자동 갱신)

1. **netlify.com** 접속 → 오른쪽 위 **Sign up** → **GitHub로 가입** (제일 쉬움)
2. 로그인 후 **Add new site → Import an existing project**
3. **Deploy with GitHub** 클릭 → 권한 허용 → 저장소 목록에서 **`oss202604/oss-homepage`** 선택
4. 설정 화면(그대로 두면 됨):
   - **Branch**: `main`
   - **Build command**: (비워둠 — 아무것도 안 적음)
   - **Publish directory**: (비워둠, 또는 점 하나 `.`)
   - ※ 정적 사이트라 빌드 과정이 없어요.
5. **Deploy site** 클릭 → 1~2분 기다리면 완료 ✅
6. 주소가 랜덤(예: `aqua-cat-1234.netlify.app`)으로 생겨요 → **이름 바꾸기**:
   - **Site configuration → Change site name → `ossohayo`** 입력
   - → 주소가 **`ossohayo.netlify.app`** 으로 바뀜

→ **끝!** 이후 Claude가 코드를 푸시하면 자동 반영돼요.

---

## 방법 B: 드래그앤드롭 (제일 간단, 단 매번 수동)

1. netlify.com 로그인 → **app.netlify.com/drop** 접속
2. 사이트 폴더(`C:\오쓰홈페이지`)를 통째로 화면에 **드래그**해서 놓기
3. 끝.
   - ⚠️ 단점: 갱신할 때마다 다시 드래그해야 해요 → **방법 A 추천**

## ⚠️ 알아둘 점

- **Supabase(회원·주문 DB)는 추가 설정 없이 바로 작동**해요. (이메일 확인 OFF 유지 — 켜면 가입 막힘)
- **이미지(메인배너·이벤트배너·카톡썸네일)는 아직 비어 있어요.** 캔바로 만들어 `img/` 폴더에 넣으면 다음 푸시 때 반영돼요. (파일명: `hero-main.png` 2480×800, `event-1.png`·`event-2.png` 1200×600, `og-cover.png` 1200×630)
- 배포는 **사장님 계정(Netlify·GitHub)으로 직접** 하셔야 해요(Claude가 대신 가입·로그인 불가). 막히는 화면 캡처해서 보내주시면 같이 풀어요.
