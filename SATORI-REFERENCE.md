# OSS 기능 사전 (Satori 레퍼런스 + OSS 적용안)

> 목적: 사장님 소유의 실사용 사이트 **Satori(satori.co.kr)**를 페이지·기능 단위로 전수 기록하여,
> OSS를 만들 때 "꺼내 쓰는 뼈대"로 삼는다. 가능한 항목은 **최신 기술/UX로 업그레이드**한다.
>
> 범례: ✅구현됨(OSS) · 🟡부분구현 · ⬜미구현 · 🔺업그레이드 제안 · 1️⃣1단계 · 2️⃣2단계
>
> 최종 업데이트: 2026-06-07 (Satori 고객 홈 조사 완료, 하위 페이지·관리자 진행 중)

## 0. 작업 메모 / 규칙
- Satori는 사장님 회사 자산. 본 문서는 **읽기 전용 조사** 결과만 기록(사토리에서 아무것도 수정하지 않음).
- 어시스턴트는 비밀번호 직접 입력 불가. 단, 조사 시점에 **이미 로그인된 세션**이 살아있어 관리자 화면도 읽기 전용 조사 가능.
- 사토리 조사 종료 시 `/member/logout` (단, 사장님 세션이므로 임의 로그아웃 금지 — 사장님 지시 시에만).
- 디자인/문구는 그대로 베끼지 않고 **기능·구조·플로우**만 참고하여 OSS 오리지널로 재구성.
- 플랫폼: Satori는 **임대형 솔루션(Hosting by Off / xeno.work)** 기반으로 추정. OSS는 자체 Supabase 스택.

### 2-5. 예상비용 계산기 (`/agency/expect/popup/1`) 🔺핵심
팝업(모달) 형태. **OSS의 정적 #fee를 동적 계산기로 업그레이드할 1순위.**

**입력:**
| 항목 | 형태 | 비고 |
|---|---|---|
| 배송센터 | 드롭다운 | 일본(항공) / (선박 등) — 배송수단별 요율 |
| 회원등급 | 드롭다운 | 레드 등 — 등급별 요율 차등(2️⃣) |
| 적용환율 | 표시값 | 예: 10.3698 (= 적용환율 1036.98/100, JPY 1엔당 원) |
| 품목분류 | [선택하기] 모달 | 통관유형·관세율 결정 |
| 총구입비용 | 숫자입력 (JPY) | |
| 예상무게 | 숫자입력 (kg) | |
| 버튼 | [초기화] [계산하기] | |

**출력:** 총구입비용 / 국제배송비 / 관·부가세 / **총예상비용**

**계산 로직(역설계):**
- 총구입비용(KRW) = 총구입비용(JPY) × 적용환율
- 국제배송비 = f(배송센터·배송수단, 무게) — 요율표 조회
- 관·부가세 = f(품목분류 통관유형, 과세표준) — 목록통관 면세한도(¥23,961≈US$150) 이하 면제, 초과/일반통관 시 관세+부가세 10%
- 총예상비용 = 총구입비용 + 국제배송비 + 관·부가세 + (구매수수료)

**품목분류 모달(`분류선택`):**
- 🔵목록통관 / 🔴일반통관 색상 구분 (통관유형이 관세 면제 여부를 좌우)
- 검색창 + [초기화] + [자주 사용하는 품목 등록] + 최근 사용/자주 사용 품목
- 대분류 예시: **패션/의류/잡화**(모든의류·가죽의류🔴·가방및지갑·속옷·수영복·신발류·보석·넥타이/스카프·모자·선글라스·시계·우산·장갑·벨트·뱀피악어🔴·티셔츠 등), **유아용품**(분유/이유식🔴·유모차·카시트·젖병·기저귀 등), **레저/스포츠**(골프용품·골프채·골프공 등) … (대분류별 수십 품목, 전체 목록은 필요 시 추가 캡처)

### 3-4. 무게별 배송비 = OSS 요율표 원본 ★ (`/adminConfig/agency-delivery-price`)
탭: **일본(항공)** / **일본(해상)**. 입력모델 = 무게구간(kg) · 증가무게(kg) · **증가배송비(JPY)** 누진.
**일본(항공) 실제 운영값(역설계):**
- 0~0.5kg = **1,000엔**(기본), 이후 ~1kg까지 0.5kg당 +150, ~2kg +150, ~10kg +150, ~15kg +250, ~20kg +300, ~40kg는 1kg당 +500
- 결과표 예: 0.5kg 1,000 / 1kg 1,150 / 2kg 1,450 / 5kg 2,350 / 10kg 3,850 / 15kg 6,350 / 20kg 9,350 / 30kg 14,350 / 40kg 19,350 (JPY)
> OSS `settings.shipping_rates`에 이 누진모델(구간·증가무게·증가요금) + 항공/해상 탭으로 구현하면 동일 결과. 예상비용 계산기와 공유.
