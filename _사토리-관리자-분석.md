## 1. 주문 상태머신 (대시보드에서 확인 — OSS는 현재 12단계뿐)

### 구매 단계 (구매대행)
- 구매요청 → 구매승인대기 → **구매결제대기** → **구매결제완료** → 송금요청 → 구매완료
- 예외: 구매보류
- (대시보드 현재 건수 예: 구매결제대기 4, 구매완료 29, 구매승인대기 2)

### 입고 단계 (창고)
- 신청접수 → 창고도착 → 일부수량입고 / 일부입고 → 입고완료 → **입고보관** → 포장/무게측정 → **(배송비)결제대기**
- (입고보관 91건 = 가장 큰 버킷)

### 출고/배송 단계
- **(배송비)결제완료** → 출고보류 → 발송대기 → 발송완료

### 예외/사고
- 확인요청 승인대기 · 확인요청 입고제외
- 사고 승인대기 · 사고 입고제외 · 폐기
- 반송/교환 요청 · 반송비 결제대기 · 반송비 결제완료 · 반송/교환 완료 · 반송/교환 불가

### 자동 지연 표시 (대시보드 우측)
- 구매 입고지연(5일), 발송지연(3일), 확인/사고 승인대기 지연(3일), 확인/사고 입고요청 지연(3일)

### 회계 요약 (대시보드 우측 하단)
- 예치금 합계, 적립금 합계, 회원수

➡️ **핵심**: 결제가 "대기 → 완료"로 명확히 갈라지고(구매결제·배송비결제 각각), 입고/출고/통관/배송 단계마다 **완료일이 따로 기록**됨.

## 4. ★주문 상세 편집창★ (확인 클릭 → 팝업) — /adminOrder/config/mode/modify/itemId/{id}/popup/

## 6. 환경설정 — /adminConfig (대분류→소분류 전수 확인)

### 6-1. 기본설정
- **기본**: 회원가입 초기레벨 / 회원레벨 명칭+아이콘 업로드(admin·BIZ·레드·다이아·골드·실버·guest, 아래→위로 상승, guest 로그인불가) / SMS사용여부·발신번호 / **구매대행 지불방법**(신한·현대… 항목추가/삭제 = 결제수단 목록) / **메모 카테고리**(확인·사고… 항목추가/삭제)
- **품목분류**: HS코드 카탈로그 207개. 대분류 칩(패션/의류/잡화·유아용품·전자제품·화장품·식품·완구 등 22개) → 품목명·적하명(영문/중문)·HS코드·통관(목록/일반)·관세율·부가세율(10%)·검색키워드·세부수정
- **관세청 고시환율**: 주차별 공식환율(USD·JPY, 전주대비 증감) 39건 — 관·부가세 계산 기준
- **송장대역대**: 택배사별 송장번호 풀(CJ대한통운 PREFIX 30 / 한진택배) 시작~종료번호·전체·사용·잔여 — 국내송장 자동발번
- **업무시간**: 지역별(일본해상·한국) 휴무요일+영업시간
- **후무설정**: 휴무일 달력 지정

### 6-2. 대행설정 (요율·수수료 = 돈 계산 핵심)
- **사이트**: 브라우저타이틀·파비콘·메타태그·사이트명·회사명·전화·메일 / 회원가입·로그인 **적립금 지급**(CASH) / **결제수단을 거래유형별로 설정**: 배송비·구매대행1차결제·반송비·예치금충전 각각 [무통장·신용카드·가상계좌] 체크 + 기본선택 / **에스크로** 사용여부·결제수단(가상계좌)
- **기본**: 부피무게 계산(가로×세로×높이 ÷6000=kg, ÷166=lb) / **배송비 할인(등급별)** 레드10%·다이아4%·골드2%·실버0% / **배송대행 수수료**(등급별 CASH, 예: 30) / **구매대행 수수료**(상품가 기준, 배송센터별 일본항공·해상, 금액구간별 JPY)
- **무게별 배송비**: 항공/해상 탭, 0.5kg 단위 증가요율(~0.5kg ¥1000, 이후 +¥150~/0.5kg, 무게 커질수록 증가폭↑) + 자동계산 미리보기표
- **서비스**(부가요금): 결제무게 계산(부피무게 vs 실무게 %) / 묶음배송료(트래킹넘버 N개까지 무료, 초과시 CASH) / 멀티박스(N박스 무료, 초과 CASH) / **보관료**(N일 무료→365일까지→이후 일마다 CASH) / 반품수수료(¥2000) / 간이통관료
- **사이트 환율**: 화폐별 현금살때환율 + 추가마진(%/CASH) = 사이트 적용환율 (예: JPY 942.71 +10% = 1036.98). [외환은행환율 적용]/[직접변경] + 변동내역
- **SMS**: 문자발송 설정

## 8.5 주문관리 운영 워크플로우 (구매관리·입고관리 — 도구형 화면들)

### 구매관리 (/adminOrder/shopping) — 사입 처리
- 하위(상태별 필터목록): 구매전체 · 승인대기 · 결제대기 · 결제완료 · 구매보류 · 구매요청 · 송금요청 · 구매완료 · 취소요청 · 구매취소
- 각 행: 트래킹넘버/오더넘버 **인라인 등록·수정**, 수량, 수수료/현지배송비, 단가/결제금액, 등록일/구매일
- 상단 일괄버튼: **선택 구매완료** (여러 건 한꺼번에 상태 전환)
- 필터에 입고지연(경과일) 추가

### 입고관리 (/adminOrder/warehouse) — 창고 작업
- 하위: 도착스캔 · 입고작업 · 신청접수 · 창고도착 · 일부수량입고 · 일부입고 · 확인/사고 입고요청 · 입고완료 · 입고보관 · 포장/무게측정 · 결제대기 · 노데이타 · 럭넘버
- **도착스캔**: 트래킹넘버 바코드 스캔 입력칸 → 창고도착 처리, 스캔내역 탭, **모바일 QR 접속**(창고 폰으로 스캔)
- **입고작업**: 상태별 목록 + 행별 사진촬영 + 상단 일괄버튼(**선택 입고 / 입고취소 / 사고요청 / 확인요청**) + 모바일 QR
- **포장/무게측정**: 목록 + 하단 일괄도구(**인수라벨 일괄출력 / 송장발급·출력 / SMS 보내기 / 휴지통**)
- 나머지(신청접수·창고도착·일부입고·입고완료·입고보관·결제대기·노데이타)는 상태별 필터목록
- **럭넘버**(랙넘버): 창고 보관위치 관리

➡️ OSS 매핑: 칸반 상태이동은 있으나 **바코드 스캔·일괄처리·라벨/송장 출력·랙위치**는 없음. 운영 핵심 도구로 신규 필요(또는 폰앱 주문도우미와 연계).

## 9. 결제관리 — /adminPayment
- 하위: 결제내역 · 적립금내역 · 예치금내역 · 예치금환불
- 결제내역(381건): 결제방법(기타/가상계좌) · 상태(입금완료/취소) · 결제명(예: "배송비 결제-JA10638" / "구매대행 1차결제-3289") + 결제주문번호 + PG거래번호 + 입금계좌 · 이름 · **예치금 · 적립금 · 쿠폰** · 결제금액 · PG수수료 · 실결제금액 · 등록일/결제완료일
- 필터: 입금자명·아이디·결제주문번호·PG거래번호·날짜·상태·사서함·결제방법·결제종류
- ➡️ **1차(구매대행)·2차(배송비)·반송비 결제가 한 목록에 통합**. 예치금/적립금/쿠폰 차감이 결제건마다 기록. 예치금/적립금 **원장(내역)**과 **예치금환불** 별도.

## 10. 정산/통계 — /adminReport
- 하위(전부 리포트 내려받기): 부가세신고 · 배송대행 · 구매대행 · 반송비 · 회원가입 · 결제금액 · 배송신청건수 · 회원별신청건수
- 각 리포트: 배송센터(항공/해상) + 기간(분기별/상하반기/날짜지정) + [내려받기](엑셀)

## 11. 구매상품 — /adminItem
- 하위: 기본설정(NEW·BEST 아이콘 업로드) · 카테고리 · 상품관리
- = 메인 노출용 추천상품/카테고리 관리

## 12. 프로모션(쿠폰) — /adminPromotion
- 하위: 배송대행쿠폰▾(배송비 회원쿠폰 · 배송비 단독쿠폰) · 구매대행쿠폰▾
- 쿠폰: 종류(쿠폰코드 / 신규가입쿠폰) · 쿠폰명 · 금액(¥) · 사용기간(제한없음/발급일+30일) · 발급개수 · 사용개수 · 발행여부 · 사용여부 · 발행내역
- ➡️ OSS는 배송비쿠폰만 있음. **구매대행쿠폰·쿠폰코드·신규가입 자동발급**으로 확장 여지.

## 13. 메루카리 — /adminMercari  ※OSS 범위 밖
- 경매계정(clientid·secret·일본주소·transaction_message) + 수동구매 + 추천키워드 = 일본 메루카리 API 자동구매 연동. (OSS 미도입)

## 14. 나머지(자명/필요시) 
- 재고현황(재고목록) · 로그(관리자 작업이력) · 휴지통(삭제주문 복원) — 표준
- 환경설정 > 대행설정(요율·수수료)·품목분류·관세고시환율·송장대역대·업무시간 — 설정값(필요시 추가 캡처)
- 회원 상세(예치금/적립금 내역·주문이력) — 회원관리 상세보기
- 배송관리·확인/사고/폐기·반송/교환 하위 = 전부 상태별 주문목록(파악된 패턴과 동일)

## ★ 결론: 13개 상위 모듈 전수 확인 완료. OSS가 따라가야 할 핵심 5축
1. **주문 상태머신 확장** (12 → ~30: 구매 6 + 입고 7 + 출고 4 + 예외 다수, 결제 대기/완료 분리)
2. **풀 주문 상세 편집창** (사입·결제수단·결제완료일·예치금/적립금/쿠폰 사용·단계별 완료일·반송)
3. **결제 통합** (1차·2차·반송비 한 목록 + 예치금/적립금 원장 + 환불)
4. **운영 도구** (바코드 도착스캔 + 일괄 입고/사진 + 인수라벨·송장 출력 + 럭넘버) — 폰앱 연계 가능
5. **설정·콘텐츠·프로모션** (결제수단·품목분류·관세환율 설정 / 게시판·후기·팝업 / 쿠폰·추천상품·배너)

## 6. OSS 구현 매핑 요약 (현재 vs 필요)
| 영역 | 사토리 | OSS 현재 | 할 일 |
|---|---|---|---|
| 주문 상태 | ~30단계, 결제 대기/완료 분리 | 12단계, 결제완료 없음 | 상태머신 확장 |
| 주문 상세 | 풀 편집창(사입·결제·예치금·반송) | 가벼운 모달 | 풀 편집창 구축 |
| 결제수단·사입 기록 | 있음 | DB만 있고 UI 없음 | 편집창에 노출 |
| 예치금/적립금 사용 | 주문에서 차감 | 잔액칸만 있음 | 차감 로직+내역 |
| 단계별 완료일 | 7종 타임라인 | status_dates jsonb 있음 | 화면 표시·기록 |
| 검색필터 | 20+ 필드 | 기본 검색 | 필터 확장 |
