## ❶ 한 줄 결론
**메루카리 상품을 사이트에 끌어오는 "공식·합법" 경로는 없다. 다들 "비공식"으로 한다.** 사장님 말씀대로 **실질 위험은 낮고 업계 표준**이지만, 약관상 회색지대이고 "누가 깨질 때 고치냐"가 핵심. → **상용 스크래퍼 API를 쓰면 유지보수 부담 없이 MVP 가능.**

## ❷ 메루카리 데이터 가져오는 방법 (전수)

### 공식 = 막힘 ✅(3표 검증)
- **Mercari Shops API**: 자기 샵 운영자 전용(Souzoh와 파트너계약 필요). 마켓 전체 상품 검색용 **아님**. (출처: engineering.mercari.com/blog/.../20221121-mercari-shops-api)
- **Mercari Ambassador(어필리에이트)**: 링크 공유 수수료 프로그램. **데이터 피드/API 아님.** (출처: help.jp.mercari.com/guide/articles/1485)
- → **공식적으로 상품목록 가져오는 길 없음.**

### 비공식 (실제로 쓰는 3가지)
| 방법 | 내용 | 난이도/유지보수 |
|---|---|---|
| **A. 상용 스크래퍼 API** ⭐ | Oxylabs·Apify·ScrapingBee 등이 메루카리 검색결과를 REST로 제공. **IP차단·CAPTCHA·JS렌더 다 대신 처리.** Oxylabs **$0.45/1000건**, 무료체험 2000건 | 🟢 **가장 낮음**(업체가 대응) |
| B. 비공식 모바일 API 직접 | `https://api.mercari.jp/v2/entities:search`(검색) `items/get`(상세). 단 요청마다 **DPoP 토큰**(P-256 키쌍 새로 생성+ES256 JWT 서명) 필요 + Cloudflare 보호 | 🔴 높음(암호서명 재현·자주 깨짐) |
| C. HTML 스크래핑+FlareSolverr | Cloudflare 우회 프록시로 검색/상세 페이지 긁기 | 🟠 중간(깨지기 쉬움) |

(출처: github marvinody/mercari, generate-mercari-jwt, wailinkyaww/mercari-api, take-kun/mercapi, oxylabs.io, apify.com, scrapingbee.com)

### 참고: Buyee 'Add to Buyee'
브라우저 확장(크롬/파폭)으로 메루카리 상품 페이지에서 버튼 눌러 Buyee 장바구니로 담는 방식. = 우리 "링크 붙여넣기"의 확장판. (출처: buyee.jp/addtobuyee)

## ❸ 법적·약관
- **메루카리 약관**: 공식 인터페이스 외 접근 금지 + 시스템 무단접근 금지 + **데이터의 외부 상업적 사용 금지**(사전 서면허가 없이). → 스크래핑·비공식API·외부표시 모두 **약관 위반**. (출처: help.jp.mercari.com/900, static.jp.mercari.com/en/global_tos)
- **한국 법**: 대법원 2021도1533(2022.5) — 경쟁사 공개 API 크롤링 **무죄**. 단 근거가 "①대상에 보안조치 없음 ②이미 공개된 데이터 ③소량 ④약관 접근제한 불명확". **메루카리는 보안조치(Cloudflare·DPoP) 있음** → 이 방어논리 약해짐. (출처: shinkim.com 뉴스레터)
- **정리**: 형사처벌 사례는 드물고 업계 표준이나, **민사 분쟁·차단·중지요청 가능성**은 존재. 리스크는 "법"보다 "차단·유지보수".

## ❹ 우리 스택(Supabase) 아키텍처
```
[손님 브라우저]  검색창 입력
      │
[Supabase Edge Function]  ← 서버사이드(CORS 해결·API키 숨김)
      │  ① Postgres 캐시 확인(같은 검색어 최근결과 있으면 즉시 반환)
      │  ② 없으면 → 상용 스크래퍼 API(Oxylabs 등) 호출
      │  ③ 결과 Postgres에 캐싱(비용·rate limit 절감)
      ▼
[결과 그리드]  사진·제목·가격 → 클릭 시 [주문서에 메루카리 URL 자동 채움]
```
- Edge Function이 CORS·키은닉·rate limit 처리. (출처: supabase.com/docs/guides/functions/cors)
- 인기 키워드는 **스케줄 함수**로 미리 캐시 갱신. (출처: supabase 스케줄/대용량 작업 문서)
- 이미지도 Edge Function 프록시로 받아 캐싱(핫링크 차단 대비).

## ❺ 추천 — 단계별
### MVP (현실적 1단계)
1. **상용 스크래퍼 API 가입**(Oxylabs/Apify, 무료체험으로 시작) — 메루카리 깨져도 **업체가 대응** = 우리 유지보수 0.
2. Supabase Edge Function 1개 = 프록시 + 캐싱.
3. 홈/주문폼에 **메루카리 검색창** → 결과 그리드 → "이 상품 구매대행" 버튼(주문서 prefill, 기존 흐름 연결).
- 비용: 사용량당 소액(검색 1000건당 ~$0.45). 트래픽 적은 초기엔 거의 무료체험으로 커버.

### 확장 (2단계+)
- 카테고리 브라우즈 / 야후옥션·라쿠텐 추가 / 인기상품 캐시 / 번역(상품명 한글).

### ❌ 비추
- **DPoP 직접 구현(B안)**: 가장 자주 깨지고 유지보수 큼. 신생 사업엔 돈 조금 써서 스크래퍼 서비스(A) 쓰는 게 합리적.
